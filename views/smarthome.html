SMARTHome

Control your household appliances remotely using wireless switches

I wanted to be able to control my heating, immersion and lighting remotely through the use of sonoff wireless switches without using the sonoff mobile app and cloud services which did not provide all the functionality I required.

Implementation

Home automation is "The Internet of Things", it simply means the way all devices or appliances are networked together to provide users with seamless control of their home equipment. The Sonoff basic is an affordable WiFi smart switch that provides users with smart home control. It is a remote control power switch that can connect to a wide range of appliances. It transmits data to an AWS global server through the WiFi Router, which enables users to remotely control all the connected appliances, via the mobile application eWeLink.

I wanted to have more control of the switch allowing me to turn on and off switches based on external inputs such as internal room temperature and external temperature in real time.
To do this I used a DHT temperature and humidity sensor connected to a Raspberry PI to read room temperature and turn on central heating when the temperature falls below a configured level. I also use the OpenWeather API to get the current weather forecast and can turn on or off the heating based on the reading.

Socket.IO enables real-time, bidirectional and event-based communication. It works on every platform, browser or device, focusing equally on reliability and speed. This enables us to comminicate with the wireless switches in real time via a Node.js server hosted on a Raspberry Pi.

Because we dont have access to the switch functionality directly, I use IFTTT (If this, then that), a software platform that connects apps, devices and services from from different developers in order to trigger one or more automations involving those apps, devices and services via applets.
